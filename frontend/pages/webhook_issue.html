Webhook Issue

webhook_issue

Overview

This response template provides troubleshooting guidance for webhook delivery problems. It helps developers debug why webhooks aren't being received or triggered properly.

Keywords

webhook callback notification not received event not triggering

Use Cases

  • Webhook endpoints not receiving events
  • Payment confirmation webhooks not triggering
  • Callback notifications missing
  • Webhook delivery failures
  • Event notifications not working

Common Scenarios

  • Endpoint Not Accessible: Webhook URL not publicly accessible
  • SSL Issues: Invalid or expired SSL certificate
  • Status Code: Endpoint not returning 200 OK
  • Firewall Blocking: Network/firewall blocking webhook requests
  • Timeout: Endpoint taking too long to respond

Response Template

Template Response

Webhook delivery issues can occur due to several reasons: (1) Verify your endpoint is publicly accessible, (2) Check if your endpoint returns a 200 status code, (3) Ensure SSL certificate is valid. You can view webhook delivery logs in your dashboard under Developers > Webhooks. Our Tech Support team will check our delivery logs and help debug the issue.

Response Approach

  1. Common Causes: List typical reasons for webhook failures
  2. Checklist: Provide step-by-step verification steps
  3. Self-Service: Point to webhook logs in dashboard
  4. Support Offer: Offer to check delivery logs
  5. Debugging: Provide troubleshooting guidance

Troubleshooting Checklist

  • Public Accessibility: Endpoint must be publicly accessible (not localhost)
  • HTTPS Required: Valid SSL certificate required
  • Status Code: Endpoint must return 200 OK
  • Response Time: Endpoint should respond within timeout window
  • Logs: Check webhook delivery logs in dashboard
  • Firewall: Ensure firewall allows incoming webhook requests

When to Use

This template is automatically selected by the RAG system when:

  • Query mentions "webhook", "callback", "notification not received"
  • Intent is classified as technical_support
  • Urgency is medium to high
  • Customer mentions webhook or event delivery issues
