Refund Request

refund_request

Overview

This response template handles refund requests and billing errors. It's used when customers need refunds for duplicate charges, incorrect charges, or billing mistakes.

Keywords

refund duplicate charge charged twice incorrect charge

Use Cases

  • Duplicate charges requiring refund
  • Incorrect billing amounts
  • Charged twice for same service
  • Billing errors needing correction
  • Refund processing requests

Common Scenarios

  • Duplicate Charge: Customer charged twice for same subscription
  • Incorrect Amount: Wrong amount charged
  • Service Not Used: Charged for unused service
  • Billing Error: System error causing incorrect charge
  • Refund Processing: Need to process refund

Response Template

Template Response

We apologize for the billing error. To process your refund: (1) Confirm the transaction ID or invoice number, (2) Specify the amount to be refunded, (3) Preferred refund method. Refunds are typically processed within 5-7 business days. Our Finance team will initiate this immediately and send you confirmation.

Response Approach

  1. Apologize: Acknowledge the billing error
  2. Required Information: List what's needed to process refund
  3. Process Steps: Outline refund process
  4. Timeline: Set expectation (5-7 business days)
  5. Team Assignment: Route to Finance team
  6. Confirmation: Promise confirmation email

Required Information

  • Transaction ID: Unique transaction identifier
  • Invoice Number: Invoice reference number
  • Amount: Exact amount to be refunded
  • Refund Method: Original payment method or alternative
  • Reason: Brief explanation of refund reason

When to Use

This template is automatically selected by the RAG system when:

  • Query mentions "refund", "duplicate charge", "charged twice"
  • Intent is classified as billing_finance
  • Urgency is medium to high
  • Customer requests refund processing

Team Assignment

Queries matching this template are routed to: Finance Team
