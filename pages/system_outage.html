System Outage

system_outage

Overview

This response template handles system outages and critical errors. It's used when customers report service disruptions, server errors, or system-wide issues affecting operations.

Keywords

down not working 500 error 504 error timeout outage

Use Cases

  • Service outages and downtime
  • HTTP 500 Internal Server Errors
  • HTTP 504 Gateway Timeout errors
  • System-wide failures
  • Critical service disruptions

Common Scenarios

  • Complete Outage: Service completely unavailable
  • Server Errors: 500 errors indicating server problems
  • Timeout Issues: 504 errors due to slow responses
  • Partial Outage: Some features not working
  • Performance Issues: System slow or unresponsive

Response Template

Template Response

We apologize for the service disruption. Our team has been notified and is investigating. You can check real-time status at status.finlink.com. For critical issues affecting operations, our Tech Support team will prioritize your case and provide updates every hour until resolved.

Response Approach

  1. Apologize: Acknowledge the service disruption
  2. Action Taken: Confirm team has been notified
  3. Status Page: Provide status page link for updates
  4. Priority Handling: Promise priority for critical issues
  5. Update Frequency: Set expectation for regular updates
  6. Resolution Commitment: Promise resolution tracking

Error Types

  • 500 Internal Server Error: Server-side application error
  • 504 Gateway Timeout: Server taking too long to respond
  • 503 Service Unavailable: Service temporarily unavailable
  • Connection Timeout: Unable to connect to server
  • Complete Outage: Service completely down

Status Communication

  • Status Page: Real-time status updates at status.finlink.com
  • Hourly Updates: Regular updates during critical outages
  • Incident Tracking: Track incident resolution progress
  • Post-Mortem: Follow-up after resolution

When to Use

This template is automatically selected by the RAG system when:

  • Query mentions "down", "not working", "500 error", "504 error", "outage"
  • Intent is classified as technical_support
  • Urgency is critical
  • Customer reports system-wide issues or errors

Team Assignment

Queries matching this template are routed to: Tech Support

Note: Critical outages may also escalate to Tech Lead
